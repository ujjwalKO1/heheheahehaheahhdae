  7. Chapter-1: Introduction to HTML and Website Development.
  8. Evolution of the Internet started in the 1970s.
  9. Early Internet supported plain text and complex command syntax.
  10. Primarily used by computer science researchers during initial phase.
  11. World Wide Web invented in 1989 by Sir Tim Berners-Lee.
  12. Berners-Lee proposed hyperlinks for easy information navigation.
Page 3
  1. HTML first released in 1993 by Berners-Lee for content formatting.
  2. Mosaic Web Browser (1992–1993) created by Andreesen and Bina.
  3. Mosaic introduced the <img> tag to display images on the web.
  4. It became the first popular “killer application” for the Internet.
  5. Web growth led to directories like Yahoo for content discovery.
  6. HTML evolved to support images, videos, and various multimedia.
  7. Modern Internet has over 5 billion users, 60% of global population.
  8. HTML and web technologies made the Internet user-friendly and commercial.
  9. HTML is the standard markup language for creating webpages and web applications.
  10. A markup language defines the layout and formatting of text documents.
  11. HTML makes webpage content interactive, structured, and dynamic.
Page 4
  1. HTML uses text-based tags to place images, tables, links, and elements.
  2. It provides the fundamental structure and content of a webpage.
  3. Web browsers interpret HTML code to display webpages to users.
  4. “Hypertext” means “text within text,” linking one webpage to another.
  5. HTML became an Internet standard in 1993.
  6. All major browsers (Chrome, Safari, Firefox, IE) support HTML.
  7. HTML ensures uniform appearance of webpages across browsers.
  8. It uses tags (e.g., <p>, <img>) and attributes to define elements.
  9. HTML defines both visual layout and semantic meaning of elements.
  10. Designed for machine interpretation, yet human-readable.
  11. Forms the foundation of all webpages, used in various applications.
  12. Understanding HTML is fundamental for modern professionals.
Page 5
  1. HTML is a simple language using tags to display text.
  2. Example tags: <b> for bold text, <i> for italicized text.
  3. HTML pages are created with text editors (like Notepad).
  4. Pages saved with a .html file extension.
  5. Basic HTML structure includes DOCTYPE, HTML section, HEAD, and BODY.
  6. DOCTYPE Declaration is the first line, informs HTML version.
  7. HTML Tag (<html> and </html>) encloses all HTML code.
  8. It marks the start and end of the HTML document.
  9. Every HTML element is nested within the <html> tags.
  10. Closing tags use a slash, e.g., </html>.
  11. Head Tag (<head> and </head>) contains metadata.
  12. Metadata in head tag is not directly visible on the page.
Page 6
  1. Head tag includes page title (browser tab), author, version info.
  2. Stylesheets for look and feel are linked within the head.
  3. JavaScript for user interactions is also included in the head.
  4. Description and keywords for search engine optimization (SEO) are there.
  5. Body Tag (<body> and </body>) holds all visible content.
  6. Visible content includes text, images, links, tables, etc.
  7. HTML elements include text, images, tables, links, and forms.
  8. Text can be bold, italicized, or have different font sizes.
  9. Information can be organized using bullet points or numbered lists.
  10. Webpages can include forms for user data entry.
  11. Specific HTML tags represent each of these webpage elements.
  12. HTML allows creating interactive webpages to collect and send data.
Page 7
  1. Paragraph Tag (<p>) defines a block of text as a paragraph.
  2. Browsers automatically add a line break before and after each paragraph.
  3. The <p> tag can contain text, images, links, or other HTML elements.
  4. Attributes like style can be used to style paragraphs (e.g., color).
  5. Headings (<h1> to <h6>) create structure within a webpage.
  6. <h1> is for the main heading, <h2> to <h6> for subheadings.
  7. Best practice: do not skip heading levels for semantic structure.
  8. Text formatting tags: <b> (bold), <u> (underline), <del>/<strike> (strikethrough).
  9. <sub> creates subscript text (e.g., H₂O).
  10. <sup> creates superscript text (e.g., x²).
  11. Subscript and superscript are commonly used in scientific or mathematical content.
Page 8
  1. Hyperlinks are a key, unique feature of HTML for navigation.
  2. <a> tag (anchor) is used to create hyperlinks.
  3. The href attribute specifies the URL of the target page or resource.
  4. Example: <a href="url">Text</a> displays a clickable link.
  5. <img> tag is used to add images to a webpage.
  6. src attribute specifies the image file location for the <img> tag.
  7. Example: <img src="image.jpg" alt="Sample Image">.
  8. Optional attributes like width and height set image dimensions.
  9. Images enhance visual appeal and attract user attention.
  10. Common image uses include logos, background images, and advertisements.
Page 9
  1. HTML provides two types of lists: unordered (<ul>) and ordered (<ol>).
  2. Unordered lists (<ul>) are bulleted lists.
  3. Ordered lists (<ol>) are numbered lists.
  4. Each list item is defined using the <li> (list item) tag.
  5. Lists can be nested to create sublists for hierarchical information.
  6. Lists are useful for displaying items like shopping lists or process steps.
  7. Tables are created using the <table> tag for structured data.
  8. <tr> tag defines a table row.
  9. <td> tag defines a table data cell for content.
  10. <th> tag defines a table header cell, typically bold and centered.
  11. Tables usually contain multiple <tr> tags, each with <td> cells.
  12. Forms are the main method for collecting user input on webpages.
Page 10
  1. Forms enable users to input text, numbers, dates, upload files.
  2. Forms support interaction via buttons, checkboxes, and radio buttons.
  3. The Google homepage is an example of a simple search form.
  4. Forms are created using the <form> tag.
  5. Inside a form, <input> tags specify different user input types.
  6. Common input types include text, number, file, radio, checkbox.
  7. submit and button are input types for actions.
  8. The type attribute defines the specific kind of input field.
  9. The action attribute specifies the server URL for processing form data.
  10. Submitted form data is transmitted to the server or stored in a database.
  11. Common forms include login, registration, search, contact, and e-commerce checkout.
Page 11
  1. Forms enable data collection, personalized experiences, and interactive web functionality.
  2. CSS stands for Cascading Style Sheets.
  3. CSS controls the appearance and layout of HTML elements.
  4. It defines how content displays on screens, print, phones, and other devices.
  5. CSS helps make webpages visually appealing with color, fonts, and design.
  6. Advantages: consistent design across multiple webpages.
  7. Saves developer time by centralizing layout control.
  8. Keeps HTML code clean by separating content (HTML) from design (CSS).
  9. Avoids repetition of inline styling and appearance inconsistencies.
  10. Reduces HTML file size and simplifies maintenance.
  11. Styles are defined in external CSS files, linked using <link> tag.
  12. Ensures uniformity across the site for elements like buttons and headers.
Page 12
  1. A website is a collection of interconnected webpages for a specific purpose.
  2. Examples: school website (bmsce.ac.in) or search engine (google.com).
  3. Websites are hosted on servers that handle user requests.
  4. Webpages within a site are linked for seamless navigation.
  5. A good website is coherent, easy to navigate, and purpose-driven.
  6. Website development process involves planning, building, and deployment.
  7. Planning (storyboards & wireframes) is essential before coding begins.
  8. Storyboards are visual representations of website flow and user interactions.
  9. They help identify user types and their expected interactions.
  10. Storyboards focus on storytelling, navigation flow, and user experience.
  11. Storyboards evolved from film and drama for planning page sequences.
  12. They serve as the first step before creating wireframes.
Page 13
  1. Wireframes are simplified visual blueprints of a webpage layout.
  2. They show space allocation, element placement, and interaction behavior.
  3. Wireframes are created without colors, fonts, or images.
  4. They are made during planning to detect usability or design issues early.
  5. Wireframes typically use boxes and lines to represent elements.
  6. Represented elements include headers, navigation menus, content sections.
  7. Calls to action (buttons/links) are also represented in wireframes.
  8. Balsamiq is a wireframing tool known for easy drag-and-drop creation.
  9. Figma is an advanced wireframing and collaboration tool.
  10. Figma was acquired by Adobe in 2022 for $20B.
  11. Figma can convert designs directly into working HTML websites.
  12. Wireframing tools help design for both web and mobile platforms.
Page 14
  1. A web server (e.g., Apache) is needed to host your website.
  2. HTML pages and related files must be in an organized folder structure.
  3. Folder structure planning can occur during wireframing.
  4. The root folder is the main directory of the website.
  5. index.html acts as the default homepage file.
  6. Consistent organization ensures easy navigation and maintenance.
  7. Common folders include /images, /fonts, /js, /css, /lib.
  8. Organized structure benefits maintenance and file management.
  9. It improves SEO (search engine optimization) and user navigation.
  10. Supports Content Management Systems (CMS) like WordPress for content.
Page 15
  1. Websites use different folders for sections or submenus.
  2. Example: /index.html for homepage, /submenu1/index.html for submenu.
  3. Folder structure mirrors navigation, organizing files by functionality.
  4. Chapter-2 introduces Computer Graphics.
  5. Computer Graphics is a branch of CS for creating/manipulating visual content.
  6. Applications include displaying text, graphs, GUIs, photo editing.
  7. Also includes video games and animated movies.
  8. Modern computer graphics often refers specifically to 3D graphics.
  9. 3D Graphics focuses on creating and managing three-dimensional visual representations.
  10. This area plays a key role within broader computer graphics.
Page 16
  1. Scope of Computer Graphics includes Image Manipulation.
  2. Digital cameras fueled growth in image editing software.
  3. Tools allow removing blemishes, cutting/pasting image parts, creating composites.
  4. Used in Film and Television for special effects.
  5. Special effects include removing wires, overlaying images, generating new frames.
  6. Applications drove the transition from analog film to digital imaging.
  7. 2D Graphics applications include tools like Microsoft Paint.
  8. Basic 2D features: drawing dots, lines, shapes, filling regions.
  9. Also includes cutting and pasting image sections.
Page 17
  1. 2D Graphics produces images by converting shapes into pixel patterns.
  2. Image Processing analyzes existing images to detect or enhance patterns.
  3. Simply: 2D Graphics creates images; Image Processing understands them.
  4. 3D Graphics overview: creating 3D virtual scenes.
  5. It converts 3D scenes into 2D images, simulating photography.
  6. 3D graphics “photographs” virtual worlds existing only as data/algorithms.
  7. Steps in 3D Graphics: Scene Creation & Encoding, then Image Production.
  8. Scene Creation is the artistic design of the 3D environment.
  9. Image Production is the computational process of generating the final image.
  10. Applications of 3D Graphics: interactive video games and animated movies.
  11. Players interact with 3D virtual worlds in video games.
  12. Animators interact with virtual environments to create movie frames.
Page 18
  1. Future of 3D Graphics: users may interact directly with virtual worlds.
  2. Potential for 3D graphics projectors and immersive storytelling.
  3. Viewers could explore or change scenarios in future applications.
  4. Advances in 3D human-machine interfaces will enable interactive experiences.
  5. Overview of 3D Graphics involves Modeling, Rendering, and Displaying.
  6. Modeling: designing and constructing a 3D scene using digital data.
  7. Analogous to building a physical set in the movie industry.
  8. The modeled scene may not exist in the real world.
  9. Rendering: converts the 3D model into a 2D image from a camera position.
  10. Involves applying mathematics and analytic geometry to project objects.
  11. Perspective Projection: objects projected along lines from a viewpoint.
  12. Perspective Projection produces images similar to human vision.
Page 19
  1. Parallel Projection: projectors are parallel, producing true object profiles.
  2. Parallel projection is typically used in engineering drawings.
  3. Image window defines the boundaries of the final image on the projection plane.
  4. Similar to a camera's viewfinder, only content within appears.
  5. Pixel computation: rendering calculates how each pixel in the image appears.
  6. Complex computation accounts for lighting, material properties, object interactions.
  7. Relies on physics and material science principles for realism.
  8. Frame buffer: computed pixels stored as a bitmap.
  9. This special memory area is called the frame buffer.
Page 20
  1. Frame buffer can be part of main memory or dedicated GPU memory.
  2. Displaying: final image from frame buffer shown immediately or stored.
  3. Real-time rendering (games, simulators) displays images instantly.
  4. Image quality limited in real-time rendering due to processing constraints.
  5. Animated movies achieve higher visual quality due to non-real-time rendering.
  6. Video game systems represent an encoded 3D virtual world.
  7. Scene repeatedly rendered into frame buffer as player interacts.
  8. Special-purpose graphics hardware (GPU) handles real-time rendering.
  9. Display shows updated images to create illusion of movement.
  10. Dedicated graphics hardware is the key difference for game consoles.
