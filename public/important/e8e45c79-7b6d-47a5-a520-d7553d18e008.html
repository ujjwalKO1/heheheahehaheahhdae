  7. Chapter 1 focuses on HTML and Website Development, including its evolution.
  8. Early Internet (1970s) supported only plain text and complex commands.
  9. Early Internet users were primarily computer science researchers.
  10. World Wide Web (1989) was invented by Sir Tim Berners-Lee.
  11. Berners-Lee proposed a hyperlink-based system for easier navigation and sharing.
Page 3
  1. HTML development started in 1993 with Berners-Lee's first version.
  2. HTML was designed for easy content formatting and sharing online.
  3. Mosaic Web Browser (1992–1993) created by Andreesen and Bina.
  4. Mosaic introduced the <img> tag for displaying images on the web.
  5. It became the "killer application" that popularized the Internet globally.
  6. Web grew with directories like Yahoo, simplifying content discovery.
  7. HTML evolved to support images, videos, and other multimedia.
  8. Modern Internet has over 5 billion users, 60% of the global population.
  9. HTML and web technologies made the Internet user-friendly and commercially powerful.
  10. HTML is the standard markup language for creating webpages.
  11. A markup language defines text document layout and formatting.
  12. HTML makes webpage content interactive, structured, and dynamic.
Page 4
  1. HTML uses text-based tags for images, tables, links, and other elements.
  2. It provides the essential structure and content of a webpage.
  3. Web browsers interpret HTML code to display webpages to users.
  4. "Hypertext" means text with links connecting different webpages.
  5. HTML became an Internet standard in 1993, ensuring broad compatibility.
  6. Supported by all major browsers: Chrome, Safari, Firefox, Internet Explorer.
  7. Ensures consistent webpage appearance across various browser platforms.
  8. Uses tags (e.g., <p>, <img>, <a>) to define elements and their attributes.
  9. Defines both visual layout and semantic meaning of webpage elements.
  10. Designed for machine interpretation but remains human-readable.
  11. HTML forms the fundamental foundation of all webpages on the Internet.
  12. Essential in education, email, banking, and modern web-based applications.
Page 5
  1. HTML is a simple language using tags to instruct browser text display.
  2. Examples: `<b>` for bold text, `<i>` for italicized text.
  3. HTML pages created using any text editor, saved with `.html` extension.
  4. All HTML pages follow a basic structure: DOCTYPE and HTML Section.
  5. DOCTYPE Declaration is the first line, informs browser of HTML version.
  6. The HTML Tag (`<html>` to `</html>`) encloses all HTML code.
  7. It marks the beginning and end of the entire HTML document.
  8. Every HTML element is nested within these root HTML tags.
  9. Closing tags use a forward slash, for example, `</html>`.
  10. Head Tag (`<head>` to `</head>`) contains metadata.
  11. Metadata is information not directly visible on the webpage.
Page 6
  1. Head tag includes page title, author info, stylesheets, and JavaScript.
  2. Also contains description and keywords for search engine optimization.
  3. Body Tag (`<body>` to `</body>`) encloses all visible webpage content.
  4. Visible content includes text, images, links, tables, and other elements.
  5. Webpages feature text, images, tables, links, and various formatting options.
  6. Information can be organized using bullet points or numbered lists.
  7. Webpages can include forms for user data entry with input elements.
  8. Each webpage element is represented using specific HTML tags.
  9. HTML enables creation of interactive webpages for data collection.
  10. The Paragraph Tag (`<p>`) defines a block of text as a paragraph.
  11. Browsers automatically add line breaks before and after each paragraph.
Page 7
  1. The `<p>` tag can contain text, images, links, or other HTML elements.
  2. Attributes can style paragraphs (e.g., `style="color: red;"`).
  3. Headings (<h1> to <h6>) create essential structure within a webpage.
  4. HTML provides six heading levels, `<h1>` being the main title.
  5. `<h2>` to `<h6>` represent subheadings of decreasing importance.
  6. Best practice: avoid skipping heading levels for logical structure.
  7. Text formatting tags: `<b>` for bold, `<u>` for underlined text.
  8. `<del>` or `<strike>` create strikethrough text effects.
  9. `<sub>` for subscript text, like in H₂O (H<sub>2</sub>O).
  10. `<sup>` for superscript text, like in x² (x<sup>2</sup>).
  11. Subscript and superscript are frequently used in scientific or mathematical content.
Page 8
  1. Hyperlinks are HTML's unique feature, allowing navigation between pages.
  2. The `<a>` tag (anchor) is used to create these hyperlinks.
  3. The `<a>` tag requires the `href` attribute to specify the target URL.
  4. Example: `<a href="URL">Link Text</a>` creates a clickable link.
  5. The `<img>` tag is used to add images to a webpage.
  6. It requires the `src` (source) attribute to specify the image file location.
  7. Example: `<img src="image.jpg" alt="Sample Image">` embeds an image.
  8. Optional attributes include `width` and `height` for image dimensions.
  9. Images enhance webpages visually and attract user attention effectively.
  10. Common image uses include logos, background images, and advertisements.
Page 9
  1. HTML provides two list types: unordered (`<ul>`) and ordered (`<ol>`).
  2. Unordered lists use bullets, while ordered lists use numbers.
  3. Each individual list item is defined using the `<li>` tag.
  4. Lists can be nested to create hierarchical sublists for organization.
  5. Useful for displaying shopping lists, places, or sequential steps.
  6. Tables display structured data, like price lists or enrollments.
  7. Tables are created using the `<table>` tag in HTML.
  8. `<tr>` defines a table row, containing `<td>` or `<th>` cells.
  9. `<td>` defines table data cells, holding content for each cell.
  10. `<th>` defines table header cells, typically bold and centered.
  11. Forms are the primary way to collect user input on webpages.
Page 10
  1. Forms enable users to input text, numbers, dates, upload files, and interact.
  2. They feature buttons, checkboxes, and radio buttons for user interaction.
  3. Example: The Google homepage is a simple form with input and buttons.
  4. Forms are created using the `<form>` tag in HTML.
  5. Inside forms, `<input>` tags specify different types of user input fields.
  6. Common input types include: text, number, file, radio, checkbox, submit.
  7. The `type` attribute defines the specific kind of input field.
  8. The `action` attribute specifies the server URL for processing form data.
  9. Submitted form data transmits to the server or stores in a database.
  10. Common form types: login, registration, search, contact, e-commerce checkout.
  11. Forms enable data collection, personalization, and interactive web functionality.
Page 11
  1. CSS stands for Cascading Style Sheets, controlling HTML appearance.
  2. CSS defines how content is displayed across screens, print, and devices.
  3. Helps make webpages visually appealing with colors, fonts, and design.
  4. CSS advantages include consistent design across multiple webpages.
  5. Saves developer time by centralizing layout and style control.
  6. Keeps HTML code clean by separating content (HTML) from design (CSS).
  7. Avoids repetitive inline styling like `<font>` or `<color>` attributes.
  8. Prevents appearance inconsistencies across different web pages.
  9. Reduces HTML file size and simplifies website maintenance tasks.
  10. Best practice: define all styles in an external CSS file.
  11. Link the external CSS file to HTML pages using the `<link>` tag.
  12. Ensures uniformity for elements like buttons, headers, and fonts site-wide.
Page 12
  1. A website is a collection of interconnected webpages for a specific purpose.
  2. Examples: school websites (bmsce.ac.in) and search engines (google.com).
  3. Websites are hosted on servers that process user requests efficiently.
  4. Webpages within a site are linked for seamless user navigation.
  5. A good website should be coherent, easy to navigate, and purpose-driven.
  6. Website development process involves planning, building, and deployment.
  7. Planning (storyboards, wireframes) is crucial before coding, like house construction.
  8. Storyboards are visual representations of a website's flow and interactions.
  9. They help identify user types and their expected interactions with the site.
  10. Storyboards focus on storytelling, navigation flow, and user experience.
  11. Evolved from film, now used in web design to plan page sequences.
  12. Storyboards serve as the initial step before creating detailed wireframes.
Page 13
  1. Wireframes are simplified visual blueprints of a webpage's layout.
  2. They show space allocation, element placement, and interaction behavior.
  3. Wireframes omit colors, fonts, or images, focusing purely on structure.
  4. Created in the planning phase to detect usability or design issues early.
  5. Typically use boxes and lines to represent headers and navigation menus.
  6. Also represent content sections and calls to action like buttons/links.
  7. Balsamiq is a wireframing tool known for easy drag-and-drop creation.
  8. Balsamiq is great for quick prototypes and gathering team feedback.
  9. Figma is an advanced wireframing and collaboration tool.
  10. Figma acquired by Adobe in 2022 for $20 billion.
  11. Figma can convert designs directly into working HTML websites.
  12. Wireframing tools aid design for both web and mobile platforms.
Page 14
  1. A web server (e.g., Apache) is essential for hosting your website.
  2. HTML pages and related files must be stored in an organized folder structure.
  3. Folder structure planning during wireframing improves organization.
  4. The root folder is the main directory of the entire website.
  5. `index.html` serves as the default homepage when no specific page is requested.
  6. Maintain consistent file organization for easier navigation and maintenance.
  7. Common folders include `/images`, `/fonts`, `/js`, and `/css`.
  8. `/lib` folder is used for common libraries within the website.
  9. Organized structure aids in easier maintenance and file management.
  10. It also contributes to better SEO and enhanced user navigation.
  11. Organized structure supports Content Management Systems (CMS) like WordPress.
Page 15
  1. Websites use subfolders to organize different sections or submenus.
  2. Example: `/index.html` for homepage, `/submenu1/index.html` for submenu.
  3. This structure mirrors navigation and keeps files organized by functionality.
  4. Chapter 2 introduces Computer Graphics as a branch of computer science.
  5. Computer Graphics uses technology to create and manipulate visual content.
  6. Applications include displaying text, graphs, and GUIs.
  7. Also used for editing photos, developing video games, and animated movies.
  8. In modern usage, "computer graphics" often refers specifically to 3D graphics.
  9. 3D Graphics focuses on creating and managing three-dimensional visual representations.
  10. This area plays a key role within the broader field of computer graphics.
Page 16
  1. Digital cameras spurred rapid growth in image editing software.
  2. Image manipulation tools remove blemishes, red-eye, and cut/paste parts.
  3. These tools create composite or entirely unreal images effectively.
  4. Computer graphics are extensively used for special effects in film and television.
  5. Applications include removing support wires and overlaying images.
  6. Transition from analog film to digital imaging was driven by these advancements.
  7. 2D Graphics applications include tools like Microsoft Paint for simple drawings.
  8. Basic 2D features: drawing dots, lines, shapes (rectangles, ovals).
  9. Also include filling regions with color and cutting/pasting image sections.
  10. Computer graphics facilitated the shift towards digital production in media.
Page 17
  1. 2D Graphics produces images by converting shapes into pixel patterns.
  2. Image Processing analyzes existing images to detect or enhance patterns (AI).
  3. Simply put: 2D Graphics creates images; Image Processing understands them.
  4. 3D Graphics focuses on creating virtual scenes and converting to 2D images.
  5. It simulates photography of virtual worlds existing only as data.
  6. Steps in 3D Graphics creation: Scene Creation & Encoding, then Image Production.
  7. Scene Creation is the artistic design of the 3D environment.
  8. Image Production is the computational process generating the final rendered image.
  9. 3D Graphics widely used in interactive video games with virtual worlds.
  10. Also used in animated movies, where animators interact with virtual environments.
Page 18
  1. Current media uses 2D displays for 3D worlds, but future aims for direct interaction.
  2. Future users may interact directly with 3D virtual worlds, not just view them.
  3. Potential for 3D graphics projectors and immersive storytelling experiences.
  4. Advances in 3D human–machine interfaces will enable interactive virtual worlds.
  5. Overview of 3D Graphics involves three main steps: Modeling, Rendering, Displaying.
  6. Modeling designs and constructs a 3D scene using digital data and algorithms.
  7. This process is analogous to building a physical set for movie production.
  8. The created 3D scene may not necessarily exist in the real world.
  9. Rendering converts the 3D model into a 2D image from a camera's perspective.
  10. Rendering applies mathematics and analytic geometry to project objects.
Page 19
  1. Perspective Projection uses projectors from a viewpoint, like human vision.
  2. Parallel Projection uses parallel projectors, creating true object profiles (engineering).
  3. The Image window defines the final image boundaries on the projection plane.
  4. Similar to a camera viewfinder, only content within the window is displayed.
  5. Pixel computation in rendering determines each image pixel's appearance.
  6. Complexity arises from accounting for lighting (intensity, direction).
  7. Material properties (shiny, soft, transparent) also affect pixel computation.
  8. Interactions among objects (shadows, reflections) are crucial for realism.
  9. Pixel computation relies on principles from physics and material science.
  10. Computed pixels are stored as a bitmap in the frame buffer memory area.
  11. Frame buffer can be main memory or dedicated GPU memory.
Page 20
  1. The final image in the frame buffer is either displayed or stored.
  2. Displayed immediately for real-time rendering in games or simulators.
  3. Stored for later use in animated movies or other non-real-time applications.
  4. Real-time rendering limits image quality due to processing constraints.
  5. Animated movies typically have higher visual quality than video games.
  6. Video game systems represent an encoded 3D virtual world with manipulation software.
  7. As a player interacts, the scene is repeatedly rendered to the frame buffer.
  8. Special-purpose graphics hardware (GPU) handles real-time rendering efficiently.
  9. The display shows updated images, creating the illusion of smooth movement.
  10. Game consoles differ from PCs by having dedicated graphics hardware built-in.
