  1. Module 5 of the 'Essentials of Information Technology' course focuses on Introduction to HTML, Website Development, and Computer Graphics.
  2. Key topics covered in Module 5 include What is HTML?, Cascading Style Sheets (CSS), Website Design and Storyboarding, and the Structure of a Website.
  3. Computer Graphics within Module 5 encompasses the scope of computer graphics and an overview of 3D graphics.
  4. Primary textbooks for the module are 'Computer Science: An Overview' (Chapter 10.1-10.4) and 'Fundamentals of Information Technology' (Chapter 12).
  5. Supplemental learning resources include multiple reference books on Information Technology and e-resources such as Swayam and NPTEL video lectures.
  6. The Early Internet (1970s) primarily supported plain text and required users to know complex command syntax for operations like sending mail.
  7. Usage of the early Internet was largely confined to computer science researchers due to its technical complexity.
  8. The World Wide Web was invented in 1989 by Sir Tim Berners-Lee.
  9. Sir Tim Berners-Lee's proposal for the World Wide Web introduced a system based on hyperlinks for easy navigation.
  10. The invention of the World Wide Web significantly simplified information sharing through the use of clickable links.
Page 2
  1. Tim Berners-Lee released the first version of HTML in 1993 to facilitate easy content formatting and sharing.
  2. The Mosaic Web Browser (1992-1993), developed by Marc Andreesen and Eric Bina, introduced the `<img>` tag and became the first popular browser, significantly popularizing the Internet.
  3. The web grew rapidly, with services like Yahoo offering directories and HTML evolving to support images, videos, and other multimedia.
  4. The modern Internet boasts over 5 billion users (60% of the global population), with HTML and web technologies making it user-friendly, accessible, and commercially powerful.
  5. HTML is the standard markup language for creating webpages and web applications, defining the layout and formatting of text documents.
  6. HTML makes webpage content interactive, structured, and dynamic, enabling the placement of images, tables, links, and other elements using text-based tags.
  7. Web browsers interpret HTML code to display the structure and content of webpages to users.
  8. The term 'hypertext' signifies 'text within text,' referring to the links that connect one webpage to another.
  9. HTML became an Internet standard in 1993 and is universally supported by all major browsers (Google Chrome, Safari, Firefox, Internet Explorer), ensuring uniform webpage appearance.
  10. HTML uses tags (e.g., `<p>`, `<img>`, `<a>`) to define elements and attributes to describe them, specifying both the visual layout and semantic meaning.
  11. HTML is designed for machine interpretation but remains simple enough for humans to read, forming the fundamental foundation of all webpages on the Internet.
  12. HTML is a fundamental technology used across education portals, email, banking, and various web-based applications, making it an essential skill for modern professionals.
Page 3
  1. HTML is a simple language that uses tags to instruct web browsers on how to display text and other content.
  2. Basic HTML tags, like `<b>` for bold and `<i>` for italic, provide fundamental text formatting capabilities.
  3. HTML pages can be created using any text editor (e.g., Notepad) and must be saved with a `.html` file extension.
  4. All HTML pages follow a basic structure comprising a DOCTYPE Declaration, an HTML section, a HEAD section, and a BODY section.
  5. The DOCTYPE Declaration is the first line of an HTML page, informing the browser about the HTML version being used.
  6. The `<html>` tag encloses all HTML code, marking the start and end of the document, with closing tags denoted by a slash (e.g., `</html>`).
  7. The `<head>` section contains metadata (information not directly visible on the page), such as the page title, author info, stylesheets, and JavaScript.
  8. The `<body>` section contains all visible content displayed to the user, including text, images, links, tables, and forms.
  9. HTML allows for the inclusion of various elements like text, images, tables, and links to other pages within a webpage.
  10. Text within HTML can be formatted with options like bold, italicized, different font sizes, and organized into bullet points or numbered lists.
  11. Webpages can include forms with input elements for user data entry, making them interactive.
  12. HTML facilitates the creation of interactive webpages that can collect user data and send it to a server for processing.
  13. The `<p>` tag is an essential HTML tag used to define a block of text, representing a paragraph.
Page 4
  1. The `<p>` tag defines a paragraph, with browsers automatically adding line breaks before and after.
  2. Paragraphs can contain various HTML elements, including text, images, and links, and can be styled using attributes like `style`.
  3. HTML provides six heading levels (`<h1>` to `<h6>`) to create structure and hierarchy within a webpage.
  4. `<h1>` signifies the main heading, while `<h2>` to `<h6>` denote subheadings of decreasing importance.
  5. A best practice for headings is to avoid skipping levels to maintain logical document structure.
  6. Text formatting tags include `<b>` for bold, `<u>` for underline, and `<del>` or `<strike>` for strikethrough text.
  7. `<sub>` (subscript) and `<sup>` (superscript) tags are used for specialized text, commonly in scientific or mathematical content.
  8. Hyperlinks, created using the `<a>` (anchor) tag, are a key HTML feature enabling navigation to other web pages or resources.
  9. The `<a>` tag requires the `href` attribute to specify the target URL for the link.
  10. The `<img>` tag is used to embed images, and it requires the `src` attribute to specify the image file's location.
  11. Optional attributes for the `<img>` tag include `width` and `height` to control image dimensions.
  12. Images significantly enhance a webpage's visual appeal and can help attract user attention.
Page 5
  1. HTML provides two types of lists: unordered (`<ul>` for bulleted) and ordered (`<ol>` for numbered).
  2. Each item in an HTML list is defined using the `<li>` (list item) tag.
  3. Lists can be nested within other lists to create sublists for hierarchical organization.
  4. Tables (`<table>`) are used to display structured data, such as price lists or enrollments.
  5. Table structure is defined by `<tr>` for rows, `<td>` for standard data cells, and `<th>` for header cells (usually bold and centered).
  6. Forms (`<form>`) are the primary method for collecting user input on webpages.
  7. Forms allow diverse user inputs, including text, numbers, dates, file uploads, buttons, checkboxes, and radio buttons.
  8. The `<input>` tag is used within a form to specify various types of user input fields.
  9. The `type` attribute of an `<input>` tag defines the specific kind of input field (e.g., `text`, `number`, `file`, `radio`, `submit`).
  10. The `action` attribute of the `<form>` tag specifies the server URL where form data is sent for processing upon submission.
  11. Submitted form data is transmitted to a server or stored in a database for further handling.
  12. Common uses for forms include login/registration, search functions, contact/feedback, and e-commerce checkout processes.
Page 6
  1. CSS (Cascading Style Sheets) is a language used to control the appearance and layout of HTML elements across various display devices.
  2. CSS enhances the visual appeal of webpages by defining aspects such as color, fonts, and overall design.
  3. Key advantages of CSS include ensuring consistent design across multiple webpages and saving developer time by centralizing layout control.
  4. CSS promotes clean HTML code by separating content from design, avoids repetitive inline styling, and simplifies website maintenance.
  5. For best practice, all CSS styles should be defined in an external file and linked to HTML pages using the <link> tag to ensure site-wide uniformity.
  6. A website is defined as a collection of interconnected webpages that serve a specific purpose, hosted on servers that handle user requests.
  7. Webpages within a website are linked via weblinks, facilitating seamless navigation for users.
  8. A good website should possess characteristics of coherence, ease of navigation, and a clear purpose.
  9. The website development process involves three main stages: planning, building, and deployment.
  10. Thorough planning, utilizing tools like storyboards and wireframes, is a crucial prerequisite before beginning the coding phase of website development.
  11. Storyboards serve as visual representations of a website's flow and user interactions, aiding in the identification of different user types and their journeys.
Page 7
  1. Storyboarding is a planning technique, evolved from film and drama, used in web design to focus on storytelling, navigation flow, and overall user experience.
  2. Storyboards are the first step in web design planning, preceding the creation of wireframes.
  3. Wireframes are simplified visual blueprints of a webpage layout that show space allocation, element placement, and interaction behavior without including colors, fonts, or images.
  4. Wireframes are created during the planning phase to detect and address usability or design issues early in the development process.
  5. Wireframes typically use basic shapes like boxes and lines to represent key website elements such as headers, navigation menus, content sections, and calls to action.
  6. Wireframing tools like Balsamiq offer easy drag-and-drop functionality for quick prototypes, while advanced tools like Figma facilitate collaboration and can even convert designs directly into working HTML.
  7. Wireframing tools are versatile, enabling designers to create layouts for both web and mobile platforms (iOS/Android).
  8. A web server (e.g., Apache) is necessary to host a website, storing its HTML pages and associated files in an organized folder structure.
  9. The planned folder structure for a website can be established during the wireframing stage to ensure better organization from the outset.
  10. The root folder serves as the main directory for a website, and the `index.html` file automatically acts as the default homepage when no specific page is requested.
  11. Maintaining a consistent and organized folder structure (e.g., `/images`, `/fonts`, `/js`, `/css`, `/lib`) is crucial for easier maintenance, efficient file management, and improved SEO.

Page 10
  1. Analytic geometry is used to project 3D objects onto a 2D projection plane.
  2. Perspective projection creates images similar to human vision by projecting objects from a common viewpoint, causing objects to appear smaller with distance.
  3. Parallel projection uses parallel projectors to produce a true profile of objects, commonly employed in engineering drawings.
  4. The image window defines the visible boundaries of the final image on the projection plane, functioning like a camera's viewfinder.
  5. Pixel computation is the rendering process of determining the final appearance of each individual pixel in an image.
  6. Pixel computation is complex, requiring consideration of lighting (intensity, direction), material properties (e.g., shininess, transparency), and interactions among objects (e.g., shadows, reflections).
  7. The frame buffer is a special memory area where computed pixels are stored as a bitmap before being displayed, often residing in dedicated graphics memory (GPU memory).
  8. Images from the frame buffer can be displayed immediately for real-time applications (like games) or stored for later use (like movies or animations).
  9. Real-time rendering often limits image quality due to processing constraints, which explains why animated movies typically have higher visual quality than video games.
  10. Video game systems utilize dedicated graphics hardware (GPU) to repeatedly render scenes in real-time, storing them in the frame buffer to create the illusion of movement based on player interaction.
  11. The presence of dedicated graphics hardware (GPU) for real-time rendering is a key distinguishing feature of game consoles and essential for modern interactive graphics.
