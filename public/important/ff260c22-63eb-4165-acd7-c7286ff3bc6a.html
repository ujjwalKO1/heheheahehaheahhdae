Important Points - Notes

Page 1
  1. Module 5 covers HTML, CSS, Website Development, and Computer Graphics essentials.
  2. Primary textbooks by Brookshear and Brylow, and Roy et al. are referenced.
Page 2
  1. Reference books and e-resources are listed for further information technology study.
  2. The Internet evolved from plain-text (1970s) to the World Wide Web (1989).
  3. Sir Tim Berners-Lee invented WWW, introducing hyperlinks for easy navigation.
Page 3
  1. HTML's first version was released in 1993, enhancing content sharing capabilities.
  2. Mosaic browser popularized the Internet by introducing the <img> tag for images.
  3. The modern Internet has over 5 billion users, made accessible through HTML.
  4. HTML is the standard markup language for creating interactive webpages and applications.
Page 4
  1. HTML uses text-based tags to structure content, interpreted by web browsers.
  2. Hypertext enables links between webpages, forming the fundamental web structure.
  3. HTML became an Internet standard in 1993, supported uniformly by major browsers.
Page 5
  1. HTML pages follow a basic structure including DOCTYPE, HTML, HEAD, and BODY tags.
  2. The <head> section contains page metadata, while <html> encloses all HTML code.
Page 6
  1. The <body> tag encloses all visible content displayed on the webpage.
  2. HTML elements like paragraphs (<p>) and lists (<ol>, <ul>) organize information.
  3. Webpages utilize HTML tags to build forms for collecting and sending user data.
Page 7
  1. Heading tags (<h1>-<h6>) structure webpage content, with <h1> as the main heading.
  2. Text formatting tags include <b> (bold), <u> (underline), and <sub> (subscript).
Page 8
  1. The <a> tag creates hyperlinks, enabling navigation between webpages globally.
  2. The <img> tag embeds images, requiring the `src` attribute for the image location.
Page 9
  1. HTML offers unordered (<ul>) and ordered (<ol>) lists, with <li> defining each item.
  2. Tables (<table>) display structured data, using <tr> for rows, <th> for headers, and <td> for cells.
  3. Forms are the primary method for collecting user input on webpages.
Page 10
  1. The <form> and <input> tags define various input types for user interaction.
  2. The `action` attribute specifies the server URL where form data is sent for processing.
Page 11
  1. CSS controls HTML element appearance and layout, ensuring consistent design.
  2. CSS separates content from design, reducing HTML file size and simplifying maintenance.
  3. External CSS files are linked to HTML pages via the <link> tag for global styling.
Page 12
  1. A website is a collection of interconnected webpages hosted on servers.
  2. Website development necessitates planning stages like storyboarding and wireframing.
  3. Storyboards visually represent website flow and user interactions, focusing on user experience.
Page 13
  1. Wireframes are simplified visual blueprints of webpage layouts, used for early issue detection.
  2. Tools like Balsamiq and Figma aid in wireframing for both web and mobile design platforms.
Page 14
  1. Websites require a web server and an organized folder structure for hosting.
  2. The `index.html` file serves as the default homepage within the website's root folder.
  3. A consistent file structure improves maintenance, SEO, and CMS integration.
Page 15
  1. Computer Graphics is a branch of computer science for creating and manipulating visual content.
  2. Applications range from graphical user interfaces (GUIs) to video games and animated movies.
  3. Modern computer graphics often refers specifically to 3D graphics and its applications.
Page 16
  1. Computer graphics enabled extensive image manipulation software and film special effects.
  2. 2D graphics applications, like Microsoft Paint, offer basic drawing functionalities.
Page 17
  1. 2D Graphics creates images, while Image Processing analyzes existing images for patterns.
  2. 3D Graphics involves creating virtual scenes and rendering them into 2D images.
  3. Key 3D graphics steps include Scene Creation and Image Production for applications like games.
Page 18
  1. Future 3D graphics aim for direct user interaction with virtual worlds, not just viewing.
  2. The 3D graphics process fundamentally includes Modeling, Rendering, and Displaying stages.
  3. Modeling designs the 3D scene; rendering converts it to a 2D image via projection.
Page 19
  1. An image window defines rendering boundaries; pixel computation is a complex task.
  2. Pixel calculation considers lighting, material properties, and object interactions like shadows.
  3. Computed pixels are stored as a bitmap in a frame buffer before final display.
Page 20
  1. Images are displayed immediately (real-time rendering) or stored for later use.
  2. GPUs handle real-time rendering in video game systems for fluid motion illusion.
  3. Dedicated graphics hardware is crucial for achieving interactive 3D experiences.
