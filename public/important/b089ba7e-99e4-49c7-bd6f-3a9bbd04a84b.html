Page 3

  1. HTML's first version released in 1993 by Berners-Lee for content formatting.
  2. Mosaic Web Browser (1992–1993) created by Marc Andreesen and Eric Bina.
  3. Mosaic browser introduced the <img> tag to display images.
  4. Became the first popular browser, a "killer application" for the Internet.
  5. Web grew, with sites like Yahoo offering directories to find content.
  6. HTML evolved to support images, videos, and various multimedia.
  7. Modern Internet has over 5 billion users, 60% of global population.
  8. HTML and web technologies made the Internet user-friendly and commercially powerful.
  9. HTML is the standard markup language for creating webpages and applications.
  10. A markup language defines the layout and formatting of text documents.
  11. HTML makes webpage content interactive, structured, and dynamic.

Page 4

  1. HTML uses text-based tags for placing images, tables, and links.
  2. HTML provides the fundamental structure and content of a webpage.
  3. Web browsers interpret HTML code to display webpages to users.
  4. "Hypertext" signifies text containing links connecting different webpages.
  5. HTML became an official Internet standard in 1993.
  6. All major browsers, including Chrome, Safari, Firefox, support HTML.
  7. Ensures webpages appear uniformly and consistently across browsers.
  8. Uses tags (e.g., <p>, <img>, <a>) and attributes to define elements.
  9. Defines both the visual layout and semantic meaning of webpage elements.
  10. Designed for machine interpretation, yet human-readable for simplicity.
  11. Forms the foundational technology for all webpages on the Internet.
  12. Essential for education portals, email, banking, and web-based applications.

Page 5

  1. HTML uses simple tags to instruct browsers on text display.
  2. Example: <b>this is bold text</b> displays "this is bold text".
  3. HTML pages can be created using any text editor like Notepad.
  4. Pages are saved with a .html file extension for browser recognition.
  5. Basic HTML structure includes DOCTYPE declaration, HTML, HEAD, and BODY sections.
  6. DOCTYPE declaration is the first line, informs browser of HTML version.
  7. All HTML code is enclosed within <html> and </html> tags.
  8. These tags mark the start and end of the entire HTML document.
  9. Every HTML element is nested inside the main <html> tags.
  10. Closing tags always use a forward slash, e.g., </html>.
  11. <head> tag encloses metadata, information not directly visible on page.

Page 6

  1. Head tag includes page title, displayed on browser tab.
  2. Contains author and version information (meta information) for the page.
  3. Links to stylesheets for managing the look and feel of the website.
  4. Includes JavaScript for enabling interactive user experiences.
  5. Stores description and keywords vital for search engine optimization (SEO).
  6. <body> tag encloses all visible content shown to the user.
  7. Body content includes text, images, links, tables, and other elements.
  8. Webpages comprise text, images, tables, and links to other pages.
  9. Text can be formatted (bold, italicized, different font sizes) via HTML.
  10. Information often organized using bullet points or numbered lists.
  11. Webpages can include forms for user data entry with various input elements.
  12. <p> tag defines a block of text as a paragraph.

Page 7

  1. Browsers automatically add a line break before and after each paragraph.
  2. The <p> tag can contain text, images, links, or other HTML elements.
  3. Attributes can style paragraphs, e.g., style="color: red;".
  4. Headings (<h1> to <h6>) create hierarchical structure within a webpage.
  5. <h1> typically serves as the main heading, often the page title.
  6. <h2> to <h6> represent subheadings of decreasing importance.
  7. Best practice: avoid skipping heading levels for logical structure.
  8. <b> tag makes text bold for emphasis.
  9. <u> tag underlines text for highlighting.
  10. <del> or <strike> tags create strikethrough text.
  11. <sub> tag creates subscript text (e.g., H₂O).
  12. <sup> tag creates superscript text (e.g., x²).

Page 8

  1. Subscript and superscript tags are common in scientific/mathematical content.
  2. Hyperlinks are a unique and key feature that define HTML.
  3. Allow users to navigate to other webpages globally with a click.
  4. The <a> tag (anchor) is used to create these hyperlinks.
  5. <a> tag requires href attribute to specify target URL.
  6. Example: <a href="https://www.example.com">Visit Example</a> for clickable link.
  7. <img> tag is used for adding images to a webpage.
  8. src (source) attribute is required to specify image file location.
  9. Example: <img src="image.jpg" alt="Sample Image"> for displaying an image.
  10. Optional width and height attributes control image dimensions.
  11. Images enhance webpages visually, attracting user attention effectively.
  12. Common uses include logos, background images, and advertisements.

Page 9

  1. HTML offers two types of lists: unordered (<ul>) and ordered (<ol>).
  2. <ul> tag creates bulleted lists for items without specific order.
  3. <ol> tag creates numbered lists for sequential or prioritized items.
  4. Each individual list item is defined using the <li> tag.
  5. Lists can be nested within each other to form sublists for organization.
  6. Useful for displaying structured information like shopping lists or steps.
  7. Tables (<table>) display structured data effectively (e.g., price lists).
  8. <tr> tag defines a table row within the table.
  9. <td> defines a table data cell, holding individual cell content.
  10. <th> defines a table header cell, usually bold and centered text.
  11. Forms (<form>) are primary method to collect user input on webpages.

Page 10

  1. Forms enable users to input text, numbers, dates, and upload files.
  2. Interactive elements like buttons, checkboxes, and radio buttons are used.
  3. Example: Google homepage features a simple search form with inputs.
  4. Forms are created using the <form> tag in HTML.
  5. Multiple <input> tags specify different types of user input within forms.
  6. Common input types: text, number, file, radio, checkbox.
  7. submit and button types are for form actions.
  8. type attribute defines the specific kind of input field.
  9. action attribute specifies server URL for form data processing.
  10. Submitted data is transmitted to the server or stored in a database.
  11. Common forms include login, registration, search, contact, and e-commerce checkouts.
  12. Forms enable data collection, personalized experiences, and interactive web functionality.

Page 11

  1. CSS stands for Cascading Style Sheets.
  2. CSS controls the appearance and layout of HTML elements.
  3. Defines how content displays across screens, print, phones, and other devices.
  4. Helps make webpages visually appealing with color, fonts, and design.
  5. Advantages: enables consistent design across multiple webpages.
  6. Saves developer time by centralizing layout control for efficiency.
  7. Keeps HTML code clean by separating content (HTML) from design (CSS).
  8. Avoids repetition of inline styling, preventing inconsistencies.
  9. Reduces HTML file size, making website maintenance easier.
  10. Best practice: define all styles in an external CSS file.
  11. Link the external CSS file to HTML pages using the <link> tag.
  12. Ensures uniformity for elements like buttons, headers, and fonts site-wide.

Page 12

  1. A website is a collection of interconnected webpages for a specific purpose.
  2. Examples include school websites (e.g., bmsce.ac.in) and search engines (google.com).
  3. Websites are hosted on servers that process user requests.
  4. Weblinks connect webpages within a site for seamless user navigation.
  5. A good website must be coherent, easy to navigate, and purpose-driven.
  6. Website development process involves planning, building, and deployment stages.
  7. Planning (storyboards & wireframes) is crucial before coding, like house construction.
  8. Storyboards are visual representations of website flow and user interactions.
  9. Help identify different user types and their expected interactions on the site.
  10. Focus on storytelling, navigation flow, and overall user experience.
  11. Evolved from film, now used in web design to plan page sequences.
  12. Serve as the essential first step before creating detailed wireframes.

Page 13

  1. Storyboards focus on telling the user's journey through the website.
  2. They are the initial blueprint before moving to detailed design.
  3. Wireframes are simplified visual blueprints showing webpage layout.
  4. Show space allocation, element placement, and interaction without styling.
  5. Created during planning phase to detect usability or design issues early.
  6. Typically use basic shapes (boxes, lines) to represent elements.
  7. Represent headers, navigation menus, content sections, and calls to action.
  8. Balsamiq is an easy drag-and-drop tool for quick wireframe prototypes.
  9. Figma is an advanced wireframing and collaboration tool, acquired by Adobe.
  10. Figma can convert designs directly into working HTML websites.
  11. Wireframing tools help design for both web and mobile platforms (iOS/Android).

Page 14

  1. A web server (e.g., Apache) is required to host a website.
  2. HTML pages and related files must be stored in an organized server folder.
  3. Folder structure planning during wireframing enhances organization.
  4. The root folder serves as the main directory for the website.
  5. index.html file acts as the default homepage when no specific page is requested.
  6. Maintaining consistent file organization is crucial for navigation and maintenance.
  7. Common folders include /images, /fonts, /js, /css, and /lib.
  8. Organized structure benefits: easier maintenance and file management.
  9. Contributes to better SEO (search engine optimization) and user navigation.
  10. Supports Content Management Systems (CMS) like WordPress for content management.

Page 15

  1. Websites use different folders for sections or submenus (e.g., /submenu1/index.html).
  2. This folder structure mirrors website navigation and organizes files by functionality.
  3. Computer Graphics is a branch using computer technology for visual content.
  4. Wide range of applications, including displaying text and creating graphs.
  5. Used for designing graphical user interfaces (GUIs) and editing photographs.
  6. Essential for developing video games and producing animated movies.
  7. In modern usage, computer graphics often specifically refers to 3D graphics.
  8. 3D Graphics focuses on creating and managing three-dimensional visual representations.
  9. This area plays a key role within the broader field of computer graphics.
  10. Module 5 introduces Computer Graphics, covered in Chapter-2.

Page 16

  1. Digital cameras fueled rapid growth in image editing software.
  2. Image manipulation tools remove blemishes, red-eye, and cut/paste parts.
  3. These tools also create composite or entirely unreal images for various purposes.
  4. Computer graphics used extensively in Film and Television for special effects.
  5. Examples: removing support wires, overlaying images, creating new frames.
  6. These applications significantly drove the shift from analog film to digital imaging.
  7. 2D Graphics applications include simple drawing tools like Microsoft Paint.
  8. Basic features: drawing dots, lines, and various shapes (rectangles, ovals).
  9. Allows filling regions with color and precisely cutting/pasting image sections.
  10. The scope of computer graphics is broad and impactful across industries.

Page 17

  1. 2D Graphics produces images by converting shapes into pixel patterns.
  2. Image Processing analyzes existing images to detect or enhance patterns (AI).
  3. In short: 2D Graphics is for creating images; Image Processing for understanding images.
  4. 3D Graphics focuses on creating 3D virtual scenes.
  5. Converts these virtual scenes into 2D images, simulating photography.
  6. 3D graphics "photographs" virtual worlds existing only as data and algorithms.
  7. Steps in 3D Graphics creation: Scene Creation/Encoding and Image Production.
  8. Scene Creation is the artistic process of designing the 3D environment.
  9. Image Production is the computational process generating the final rendered image.
  10. Applications of 3D Graphics: interactive video games and animated movies.
  11. Players interact with 3D virtual worlds in video games.
  12. Animators interact with virtual environments to create movie frames.

Page 18

  1. Future of 3D Graphics involves direct interaction with virtual worlds.
  2. Current media uses 2D displays for 3D worlds; future may allow direct engagement.
  3. Potential for 3D graphics projectors and immersive storytelling experiences.
  4. Users could explore or change scenarios within these immersive environments.
  5. Advances in 3D human–machine interfaces will enable such interactive experiences.
  6. Overview of 3D Graphics process: Modeling → Rendering → Displaying.
  7. Modeling involves designing and constructing a 3D scene using data and algorithms.
  8. Analogous to building a physical set in the film industry.
  9. The created 3D scene may not exist in the real world.
  10. Rendering converts the 3D model into a 2D image from a camera view.
  11. Applies mathematics and analytic geometry to project objects onto a plane.
  12. Perspective Projection uses a common viewpoint, producing human-vision like images.

Page 19

  1. Parallel Projection uses parallel projectors, showing true object profiles (engineering).
  2. The image window defines the boundaries of the final image on the projection plane.
  3. This is similar to a camera's viewfinder, displaying only content within its frame.
  4. Pixel computation in rendering determines how each pixel in the image appears.
  5. This complex process accounts for lighting intensity, direction, and material properties.
  6. It also considers interactions among objects, such as shadows and reflections.
  7. The process relies heavily on principles derived from physics and material science.
  8. Computed pixels are stored as a bitmap in a special memory area called the frame buffer.
  9. The frame buffer can be part of main memory or dedicated graphics memory (GPU memory).
  10. This storage mechanism is crucial for image display and manipulation.

Page 20

  1. The final image in the frame buffer is either displayed or stored.
  2. Displayed immediately for real-time applications like games or simulators.
  3. Stored for later use in productions like movies or animations.
  4. Real-time rendering often limits image quality due to processing constraints.
  5. Consequently, animated movies boast higher visual quality than video games.
  6. Example: A video game system represents an encoded 3D virtual world.
  7. Game software manipulates the virtual world based on player interaction.
  8. Scene repeatedly rendered and stored in the frame buffer as player interacts.
  9. Special-purpose graphics hardware (GPU) handles real-time rendering efficiently.
  10. The display shows updated images, creating the illusion of smooth movement.
  11. Difference between game console and PC lies in dedicated graphics hardware.
