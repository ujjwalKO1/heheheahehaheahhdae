Page 2
  1. Tim Berners-Lee released the first version of HTML in 1993 to enable easy content formatting and sharing.
  2. The Mosaic Web Browser (1992-1993), created by Marc Andreesen and Eric Bina, was the first popular browser and introduced the `<img>` tag for displaying images, significantly popularizing the Internet.
  3. HTML evolved to support images, videos, and various multimedia, enhancing web content richness.
  4. HTML and related web technologies have made the Internet user-friendly, accessible, and commercially powerful, now reaching over 5 billion users globally.
  5. HTML is the standard markup language for creating webpages and web applications, defining the layout and formatting of text documents.
  6. HTML makes webpage content interactive, structured, and dynamic, allowing the placement of elements like images, tables, and links using text-based tags.
  7. HTML provides the core structure and content of a webpage, which web browsers then interpret and display to users.
  8. The term 'Hypertext' refers to 'text within text,' signifying the use of links to connect different webpages.
  9. HTML became an Internet standard in 1993 and is universally supported by all major browsers (e.g., Google Chrome, Safari, Firefox), ensuring consistent webpage appearance.
  10. HTML uses tags (e.g., `<p>`, `<img>`, `<a>`) to define elements and attributes to describe them, specifying both visual layout and semantic meaning.
  11. HTML is designed for machine interpretation but is simple enough for humans to read and understand.
  12. HTML forms the fundamental foundation of virtually all webpages and web-based applications on the Internet.
  13. HTML is widely applied across diverse sectors, including education portals, email, banking, and various web applications.
  14. Understanding HTML is considered a fundamental and essential technology for every modern professional.
Page 3
  1. HTML is a simple language that uses tags to instruct web browsers on how to display text and other content.
  2. Tags like `<b>` and `<i>` are fundamental for basic text formatting, such as making text bold or italicized.
  3. HTML pages are plain text files that can be created using any text editor (e.g., Notepad) and must be saved with a `.html` extension.
  4. All HTML documents adhere to a basic structure comprising a DOCTYPE declaration, followed by the `<html>` section which encapsulates `<head>` and `<body>` tags.
  5. The DOCTYPE declaration is the very first line of an HTML page, informing the browser about the specific HTML version the document follows.
  6. The `<html>` tag encloses all HTML code, marking the start and end of the entire HTML document, with closing tags using a forward slash (e.g., `</html>`).
  7. The `<head>` tag contains metadata—information about the page that is not directly visible to the user on the webpage itself.
  8. Metadata within the `<head>` can include the page title (displayed on the browser tab), author information, links to stylesheets for styling, JavaScript for interactivity, and descriptions/keywords for search engine optimization (SEO).
  9. The `<body>` tag contains all the visible content shown to the user, such as text, images, links, tables, and other interactive elements.
  10. HTML elements represent diverse webpage content, including formatted text, images, tables, links to other pages, bullet points, numbered lists, and forms for user data entry.
  11. HTML enables the creation of interactive webpages that can collect user data through various input elements and send it to a server for processing.
  12. The `<p>` tag is an essential HTML tag specifically used to define a block of paragraph text.
Page 4
  1. The `<p>` tag defines paragraphs, automatically adding line breaks, and can contain various HTML elements and styling attributes.
  2. HTML provides six heading levels (`<h1>` to `<h6>`) to structure webpage content, with `<h1>` for the main title and subsequent tags for decreasing importance.
  3. Best practice for headings is not to skip levels (e.g., use `<h2>` under `<h1>`, not `<h3>` directly).
  4. Basic text formatting includes `<b>` for bold, `<u>` for underline, and `<del>` or `<strike>` for strikethrough text.
  5. The `<sub>` (subscript) and `<sup>` (superscript) tags are used for specialized text formatting, often in scientific or mathematical content.
  6. Hyperlinks, created with the `<a>` (anchor) tag, are fundamental for navigating between webpages and external resources.
  7. The `<a>` tag requires the `href` attribute to specify the URL of the target page or resource.
  8. The `<img>` tag is used to embed images into a webpage.
  9. The `<img>` tag requires the `src` attribute to specify the location of the image file.
  10. The `alt` attribute for `<img>` provides alternative text, crucial for accessibility and when an image fails to load.
  11. Optional attributes like `width` and `height` can be used with the `<img>` tag to control image dimensions.
  12. Images significantly enhance the visual appeal of webpages and help attract user attention.
Page 5
  1. HTML uses `<ul>` for unordered (bulleted) lists and `<ol>` for ordered (numbered) lists.
  2. Each individual item within any list type is defined using the `<li>` (list item) tag.
  3. Lists can be nested within other lists to create sublists for hierarchical organization.
  4. Tables are used to display structured data, such as price lists, enrolments, or memberships.
  5. The `<table>` tag creates a table, with `<tr>` defining a table row.
  6. `<td>` defines a table data cell (the content of each cell), while `<th>` defines a table header cell, which is usually bold and centered.
  7. Forms are the main way to collect user input on webpages, allowing for text, numbers, files, and selections.
  8. Forms are created using the `<form>` tag, which encloses all input elements.
  9. The `<input>` tag specifies different types of user input within a form, with its behavior defined by the `type` attribute.
  10. Common `input` types include `text`, `number`, `file`, `radio`, `checkbox`, `submit`, and `button`.
  11. The `action` attribute of the `<form>` tag specifies the server URL where the form data is sent for processing.
  12. Upon submission, form data is transmitted to a server or stored in a database for further use.
  13. Examples of common forms include Login/Registration, Search, Contact/Feedback, and E-commerce checkout forms.
Page 6
  1. CSS (Cascading Style Sheets) is fundamental for controlling the appearance and layout of HTML elements, dictating how content is displayed across diverse devices.
  2. CSS significantly enhances the visual appeal of webpages through the application of colors, fonts, and overall design.
  3. Key advantages of CSS include ensuring consistent design across multiple pages, saving developer time through centralized style control, maintaining clean HTML by separating content from design, and reducing HTML file size for easier maintenance.
  4. For best practice, all CSS styles should be defined in an external CSS file, which is then linked to HTML pages using the `<link>` tag.
  5. Utilizing external CSS ensures uniformity across an entire website, guaranteeing that elements like buttons, headers, and fonts remain consistent.
  6. A website is a collection of interconnected webpages serving a specific purpose, hosted on servers that process user requests.
  7. Webpages within a site are interconnected via weblinks, enabling seamless navigation for users.
  8. A good website is characterized by coherence, ease of navigation, and a clear purpose.
  9. The website development process encompasses three main stages: planning, building, and deployment.
  10. Planning, which includes creating storyboards and wireframes, is an essential initial step in website development, comparable to architectural planning before construction.
  11. Storyboards are visual representations illustrating a website's flow and user interactions, crucial for identifying different user types and their journeys through the site.
Page 7
  1. Storyboards are the initial planning step in web design, focusing on storytelling, navigation flow, and overall user experience, evolving from film and drama.
  2. Wireframes are simplified visual blueprints of a webpage layout, created during the planning phase to detect usability or design issues early.
  3. Wireframes show space allocation, element placement, and interaction behavior using only boxes and lines, without colors, fonts, or images.
  4. Common elements represented in wireframes include headers, navigation menus, content sections, and calls to action.
  5. Balsamiq is a wireframing tool known for easy drag-and-drop creation, suitable for quick prototypes and team feedback.
  6. Figma is an advanced, collaborative wireframing tool that can convert designs directly into working HTML websites, also supporting both web and mobile platforms.
  7. A web server (e.g., Apache) is necessary to host a website, storing HTML pages and related files in an organized folder structure.
  8. The website's folder structure can be planned during the wireframing stage for better overall organization.
  9. The root folder is the main directory of a website, with `index.html` serving as the default homepage displayed when no specific page is requested.
  10. Maintaining a consistent and organized folder structure is crucial for easy navigation, maintenance, and file management.
  11. Common subfolders for website organization include `/images`, `/fonts`, `/js` (JavaScript files), `/css` (stylesheets), and `/lib` (common libraries).
  12. Benefits of an organized folder structure include easier maintenance, improved file management, and better SEO.
Page 8
  1. Website navigation often uses a folder structure (e.g., /submenu1/index.html) to organize content and mirror the site's layout.
  2. Computer Graphics is a branch of computer science focused on creating and manipulating visual content.
  3. Applications of Computer Graphics are extensive, including displaying text, creating graphs/charts, designing GUIs, editing photos, developing video games, and producing animated movies.
  4. In modern usage, the term 'computer graphics' often specifically refers to 3D graphics.
  5. 3D Graphics focuses on creating and managing three-dimensional visual representations and is a key part of the broader field of computer graphics.
  6. The advent of digital cameras significantly boosted the development of software for image manipulation.
  7. Image manipulation tools allow for tasks such as removing blemishes, cutting/pasting image parts, and creating composite or unreal images.
  8. Computer graphics are crucial for special effects in film and television, including removing support wires, overlaying images, or creating new frames.
  9. The applications of computer graphics in film and television drove the transition from analog film to digital imaging.
  10. 2D graphics applications, such as Microsoft Paint, provide tools for simple drawings.
  11. Basic features of 2D graphics include drawing dots, lines, and shapes (rectangles, ovals), filling regions with color, and cutting/pasting elements.
Page 9
  1. 2D Graphics focuses on creating images by converting shapes into pixel patterns, whereas Image Processing analyzes existing images to detect or enhance patterns, often used in AI.
  2. 3D Graphics is primarily concerned with creating 3D virtual scenes and converting them into 2D images, simulating photography of virtual worlds that exist only as data and algorithms.
  3. The creation of 3D graphics involves two main stages: artistic 'Scene Creation & Encoding' (designing the environment) and computational 'Image Production' (generating the rendered image).
  4. 3D graphics are widely applied in interactive video games, where players interact with virtual worlds, and animated movies, where animators create frames within virtual environments.
  5. The future of 3D graphics is projected to move beyond 2D displays of 3D worlds towards direct user interaction with 3D virtual environments.
  6. Future advancements in 3D graphics include the potential for 3D projectors, immersive storytelling where viewers can explore or change scenarios, and more advanced human-machine interfaces.
  7. The overall process of creating and displaying 3D images follows a core paradigm: Modeling → Rendering → Displaying.
  8. Modeling is the initial step in 3D graphics, involving the design and construction of a 3D scene using digitally encoded data and algorithms, analogous to building a physical movie set.
  9. Modeled 3D scenes are not limited by reality and can represent environments that do not exist in the physical world.
  10. Rendering is the computational process that converts a 3D model into a 2D image, as viewed from a specific virtual camera position, heavily relying on mathematical principles.
Page 10
  1. Analytic geometry is used to project 3D objects onto a 2D projection plane.
  2. Perspective projection projects objects from a common viewpoint, mimicking human vision.
  3. Parallel projection uses parallel projectors to produce a true profile of objects, commonly used in engineering drawings.
  4. The 'image window' defines the boundaries of the final image on the projection plane, similar to a camera's viewfinder.
  5. Pixel computation is a complex rendering process that determines how each pixel appears in the final image.
  6. Pixel computation accounts for lighting (intensity, direction), material properties (e.g., shiny, transparent), and object interactions (e.g., shadows, reflections).
  7. The process of pixel computation relies heavily on principles from physics and material science.
  8. The 'frame buffer' is a special memory area (main or GPU memory) where computed pixels are stored as a bitmap.
  9. Rendered images can be displayed immediately for real-time applications (games) or stored for later use (movies).
  10. Real-time rendering often limits image quality due to processing constraints, explaining quality differences between games and animated movies.
  11. In video games, player interaction triggers repeated rendering of the 3D virtual world into the frame buffer, with updated images creating the illusion of movement.
  12. Special-purpose graphics hardware (GPU) is essential for handling the demands of real-time rendering in systems like video games.
  13. The presence of dedicated graphics hardware (GPU) is a key differentiator for game consoles compared to standard PCs, enabling high-performance rendering.
