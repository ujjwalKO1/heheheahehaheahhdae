Page 2
  1. Additional reference books for Information Technology are listed.

  3. Chapter 1 introduces HTML and the historical evolution of website development.
  4. The Early Internet (1970s) was text-based, requiring complex command syntax.
  5. It was primarily utilized by computer science researchers for data exchange.
  6. Sir Tim Berners-Lee invented the World Wide Web in 1989.
  7. Berners-Lee's system was based on hyperlinks for easy navigation and sharing.
  8. The World Wide Web significantly simplified information access through clickable links.
  9. This marked a pivotal shift from command-line interfaces to graphical navigation.
  10. The module covers both the history and contemporary relevance of web technologies.
Page 3
  1. HTML's first version was released by Berners-Lee in 1993 for content formatting.
  2. The Mosaic Web Browser (1992-1993) was created by Marc Andreesen and Eric Bina.
  3. Mosaic introduced the `<img>` tag, enabling image display on webpages.
  4. It became the first popular browser, accelerating Internet adoption as a 'killer application'.
  5. The web grew rapidly, with HTML evolving to include images, videos, and multimedia.
  6. Today, the Modern Internet has over 5 billion users globally.
  7. HTML and web technologies made the Internet user-friendly, accessible, and commercially powerful.
  8. HTML is the standard markup language for creating webpages and web applications.
  9. A markup language defines the layout and formatting of text documents.
  10. HTML makes webpage content interactive, structured, and dynamic for users.
Page 4
  1. HTML uses text-based tags to place images, tables, links, and other elements.
  2. It provides the fundamental structure and content of any webpage.
  3. Web browsers interpret HTML code to render and display webpages to users.
  4. Hypertext means 'text within text,' referring to links connecting web pages.
  5. HTML became an Internet standard in 1993, gaining widespread adoption.
  6. All major browsers, including Chrome, Safari, and Firefox, support HTML uniformly.
  7. This ensures webpages appear consistently across different browsing platforms.
  8. HTML uses tags (e.g., `<p>`, `<img>`, `<a>`) and attributes for element description.
  9. It defines both the visual layout and the semantic meaning of webpage elements.
  10. HTML forms the underlying foundation of all webpages on the Internet today.
  11. Understanding HTML is a fundamental skill for any modern professional.
Page 5
  1. HTML is a simple language that uses tags to instruct browsers on text display.
  2. Examples include `<b>` for bold text and `<i>` for italicized text.
  3. HTML pages can be created with any text editor and saved with a `.html` extension.
  4. The basic structure of an HTML page includes DOCTYPE, HTML, HEAD, and BODY sections.
  5. The DOCTYPE Declaration is the first line, informing the browser of the HTML version.
  6. The `<html>` tag encloses all HTML code, marking the document's start and end.
  7. All HTML elements are nested hierarchically within the `<html>` tags.
  8. The `<head>` tag contains metadata, which is not directly visible on the page.
  9. Head content includes the page title, author info, stylesheets, and JavaScript.
  10. It also holds description and keywords for search engine optimization (SEO).
Page 6
  1. The `<head>` tag includes metadata like page title, stylesheets, and JavaScript links.
  2. The `<body>` tag encloses all visible content displayed to the user on the webpage.
  3. Visible content includes text, images, links, tables, and various form elements.
  4. HTML allows text formatting such as bolding, italicizing, and different font sizes.
  5. Information can be organized using bullet points or numbered lists for clarity.
  6. Webpages can include forms for user data entry, enhancing interactivity.
  7. Each specific element on a webpage is represented using distinct HTML tags.
  8. HTML enables interactive webpages that can collect and send data to a server.
  9. The `<p>` tag defines a block of text as a paragraph, adding structure.
  10. Understanding these core elements is crucial for effective webpage construction.
Page 7
  1. Browsers automatically add line breaks before and after each paragraph (`<p>`).
  2. The `<p>` tag can contain various elements like text, images, and links.
  3. Paragraphs can be styled using attributes, for instance, changing text color.
  4. Heading tags (`<h1>` to `<h6>`) create a hierarchical structure within a webpage.
  5. `<h1>` represents the main heading, while `<h2>` to `<h6>` denote subheadings.
  6. Best practice advises against skipping heading levels for better structure.
  7. Text formatting tags include `<b>` (bold), `<u>` (underline), and `<del>` (strikethrough).
  8. The `<sub>` tag creates subscript text, seen in chemical formulas like H₂O.
  9. The `<sup>` tag creates superscript text, used for exponents like x².
  10. Subscript and superscript tags are commonly used for scientific or mathematical content.
Page 8
  1. Hyperlinks are a key HTML feature, enabling navigation between webpages.
  2. The `<a>` tag, short for anchor, is used to create these hyperlinks.
  3. The `<a>` tag requires the `href` attribute to specify the target URL.
  4. Example: `<a href="https://www.example.com">Visit Example</a>` creates a clickable link.
  5. The `<img>` tag is used to embed images directly into a webpage.
  6. It requires the `src` (source) attribute to specify the image file location.
  7. Example: `<img src="image.jpg" alt="Sample Image">` displays an image.
  8. Optional attributes like `width` and `height` can control image dimensions.
  9. Images enhance the visual appeal of webpages and capture user attention.
  10. Common uses for images include logos, background visuals, and advertisements.
Page 9
  1. HTML provides `<ul>` for unordered (bulleted) lists and `<ol>` for ordered (numbered) lists.
  2. Each individual list item is defined using the `<li>` (list item) tag.
  3. Lists can be nested within one another to create more complex sublists.
  4. Lists are highly useful for displaying organized information like shopping lists.
  5. Tables are used to display structured data, such as price lists or memberships.
  6. The `<table>` tag is used to create a table structure on a webpage.
  7. Within a table, `<tr>` defines a table row, structuring content horizontally.
  8. `<td>` defines a table data cell, holding the actual content.
  9. `<th>` defines a table header cell, typically bold and centered for clarity.
  10. Forms are introduced as the primary mechanism for collecting user input on webpages.
Page 10
  1. Forms are the main method for collecting diverse user input on webpages.
  2. Inputs include text, numbers, dates, file uploads, buttons, and checkboxes.
  3. The Google homepage serves as a simple example of a text input form.
  4. Forms are created using the `<form>` tag, which encloses all input elements.
  5. Multiple `<input>` tags define different types of user input fields within a form.
  6. Common input types include `text`, `number`, `file`, `radio`, and `submit`.
  7. The `type` attribute specifies the exact kind of input field to be rendered.
  8. The `action` attribute specifies the server URL where form data is sent for processing.
  9. Submitted form data is transmitted to the server or stored in a database.
  10. Forms enable personalized experiences, data collection, and interactive web functionality.
Page 11
  1. CSS (Cascading Style Sheets) controls the appearance and layout of HTML elements.
  2. It defines how webpage content is displayed across screens, print, and mobile devices.
  3. CSS helps make webpages visually appealing through colors, fonts, and design.
  4. A key advantage is achieving consistent design across multiple webpages efficiently.
  5. CSS saves developer time by centralizing layout control in style sheets.
  6. It separates content (HTML) from design (CSS), keeping HTML code clean and organized.
  7. CSS avoids repetitive inline styling, preventing inconsistencies in appearance.
  8. Using CSS reduces HTML file size and simplifies overall website maintenance.
  9. Best practice involves defining all styles within an external CSS file.
  10. Linking an external CSS file ensures uniformity across the site for elements like buttons.
Page 12
  1. A website is a collection of interconnected webpages serving a specific purpose.
  2. Websites are hosted on servers that process and handle user requests.
  3. Webpages within a site are linked via weblinks for seamless user navigation.
  4. A well-designed website should be coherent, easy to navigate, and purpose-driven.
  5. Website development follows a systematic process of planning, building, and deployment.
  6. Planning, including storyboards and wireframes, is crucial before any coding begins.
  7. Storyboards are visual representations of the website's flow and user interactions.
  8. They help teams identify different user types and their expected interactions.
  9. Storyboards focus on storytelling, navigation flow, and the overall user experience.
  10. Evolved from film, storyboards serve as the initial planning step before wireframes.
Page 13
  1. Wireframes are simplified visual blueprints illustrating a webpage's layout.
  2. They show space allocation, element placement, and interaction behavior, without styling.
  3. Wireframes are created during the planning phase to detect usability or design issues early.
  4. Typically, they use boxes and lines to represent headers, menus, and content sections.
  5. Wireframing tools include Balsamiq, known for easy drag-and-drop prototypes.
  6. Figma is an advanced wireframing and collaboration tool, recently acquired by Adobe.
  7. Figma offers the capability to convert design mockups directly into HTML websites.
  8. Wireframing tools support the design process for both web and mobile platforms.
  9. These tools are essential for visualizing and refining website structure.
  10. They help ensure user experience is optimized before development commences.
Page 14
  1. A web server, such as Apache, is necessary to host your website files.
  2. HTML pages and related files must be stored in an organized folder structure on the server.
  3. This folder structure should ideally be planned during the wireframing stage.
  4. The root folder serves as the main directory for the entire website.
  5. The `index.html` file acts as the default homepage, loaded when no specific page is requested.
  6. Maintaining a consistent folder organization aids easy navigation and maintenance.
  7. Common subfolders include `/images`, `/fonts`, `/js` (JavaScript), `/css`, and `/lib`.
  8. An organized structure facilitates easier maintenance and efficient file management.
  9. It also contributes to better SEO (search engine optimization) and user navigation.
  10. Well-organized structures support Content Management Systems (CMS) like WordPress.
Page 15
  1. Websites often use subfolders to mirror navigation and organize content by functionality.
  2. Example: `/submenu1/index.html` represents a distinct section of the site.
  3. Chapter 2 introduces the fundamental concepts of Computer Graphics.
  4. Computer Graphics is a computer science branch creating and manipulating visual content.
  5. Applications include displaying text, creating graphs, and designing graphical user interfaces (GUIs).
  6. It also encompasses photo editing, video game development, and animated movie production.
  7. In modern usage, 'computer graphics' often specifically refers to 3D graphics.
  8. 3D Graphics focuses on creating and managing three-dimensional visual representations.
  9. This area plays a key role within the broader field of computer graphics.
  10. The module explores the foundational principles and applications of computer graphics.
Page 16
  1. Digital cameras spurred the rapid growth of image manipulation software.
  2. Image editing tools allow removing blemishes, cutting/pasting, and creating composite images.
  3. Computer graphics are extensively used in film and television for special effects.
  4. These applications drove the transition from analog film to digital imaging.
  5. 2D Graphics applications, like Microsoft Paint, enable simple drawings.
  6. Basic 2D features include drawing dots, lines, shapes, and filling regions with color.
  7. 2D Graphics primarily produces images by converting shapes into pixel patterns.
  8. Image Processing analyzes existing images to detect or enhance patterns, often in AI.
  9. In essence, 2D Graphics focuses on image creation, while Image Processing focuses on understanding.
  10. The scope of computer graphics ranges from simple tools to complex media production.
Page 17
  1. 3D Graphics focuses on creating virtual 3D scenes and converting them into 2D images.
  2. It simulates photography of virtual worlds defined by data and algorithms.
  3. 3D graphics creation involves Scene Creation and Encoding, an artistic design process.
  4. Image Production is the computational process of generating the final rendered image.
  5. Major applications of 3D graphics include interactive video games.
  6. Animated movies utilize 3D graphics, with animators interacting with virtual environments.
  7. The future of 3D graphics envisions direct user interaction with virtual worlds.
  8. Potential advancements include 3D projectors and immersive storytelling experiences.
  9. Users may eventually explore and modify scenarios within 3D virtual environments.
  10. Advances in 3D human-machine interfaces are critical for these interactive experiences.
Page 18
  1. The 3D graphics process involves three main steps: Modeling, Rendering, and Displaying.
  2. Modeling designs and constructs a 3D scene using digitally encoded data and algorithms.
  3. This process is analogous to building a physical set for a film production.
  4. The created 3D scene may not necessarily exist in the real world.
  5. Rendering converts the 3D model into a 2D image from a specific camera position.
  6. Rendering employs mathematics and analytic geometry to project objects onto a plane.
  7. Perspective Projection projects objects from a viewpoint, mimicking human vision.
  8. Parallel Projection uses parallel projectors, providing true object profiles for engineering.
  9. Understanding these projection types is crucial for transforming 3D models into 2D images.
  10. The entire paradigm enables the creation of realistic or stylized visual content.
Page 19
  1. The image window defines the boundaries of the final image on the projection plane.
  2. Similar to a camera viewfinder, only content within the window appears in the image.
  3. Pixel computation is the complex process of determining how each pixel should appear.
  4. This computation accounts for lighting, including intensity and direction.
  5. It also considers material properties like shininess, softness, or transparency.
  6. Furthermore, it calculates interactions among objects, such as shadows and reflections.
  7. Pixel computation relies heavily on principles derived from physics and material science.
  8. The computed pixels are stored as a bitmap in a special memory area called the frame buffer.
  9. The frame buffer can be part of the main system memory or dedicated GPU memory.
  10. Accurate pixel computation is vital for achieving realistic and detailed 3D renderings.
Page 20
  1. The final image in the frame buffer is either displayed immediately or stored for later use.
  2. Immediate display is characteristic of real-time rendering, used in games and simulators.
  3. Images are stored for later use in applications like animated movies or static visualizations.
  4. Real-time rendering typically limits image quality due to processing constraints.
  5. Consequently, animated movies often achieve higher visual quality than video games.
  6. A video game system represents an encoded 3D virtual world manipulated by software.
  7. As the player interacts, the scene is repeatedly rendered and stored in the frame buffer.
  8. Special-purpose graphics hardware (GPU) handles the demands of real-time rendering.
  9. The display rapidly updates images, creating the illusion of continuous movement.
  10. Dedicated graphics hardware distinguishes game consoles from standard personal computers.
