Page 4

  1. HTML allows placement of images, tables, and links using text-based tags.
  2. It primarily provides the structure and content for a webpage.
  3. Web browsers interpret HTML code to accurately display webpages.
  4. Hypertext signifies 'text within text,' facilitating inter-page links.
  5. HTML was established as an Internet standard in 1993.
  6. All major browsers (Chrome, Safari, Firefox) consistently support HTML.
  7. HTML uses tags (e.g., <p>, <img>) to define elements and attributes.
  8. It defines both the visual layout and semantic meaning of elements.
  9. HTML is machine-interpretable but designed for human readability.
  10. It forms the foundational technology for all internet webpages.
  11. HTML is crucial for education portals, email, banking, and web applications.
  12. Understanding HTML is fundamental for every modern professional.

Page 5

  1. HTML is a simple language using tags to instruct browser display.
  2. Example tags include `<b>` for bold and `<i>` for italicized text.
  3. HTML pages are created using text editors and saved as .html files.
  4. Every HTML page follows a basic structure: DOCTYPE and HTML Section.
  5. The HTML Section contains both the HEAD and BODY components.
  6. DOCTYPE Declaration is the first line, specifying the HTML version.
  7. All HTML code must be enclosed within `<html>` and `</html>` tags.
  8. `<html>` and `</html>` mark the start and end of the document.
  9. Closing tags in HTML use a forward slash, like `</html>`.
  10. The `<head>` tag contains metadata not directly visible on the page.

Page 6

  1. Head tag contains page title, author info, stylesheets, JavaScript, and SEO keywords.
  2. The `<body>` tag encloses all visible content displayed to the user.
  3. Visible content includes text, images, links, and tables.
  4. Webpages contain diverse elements like text, images, tables, and links.
  5. Text can be formatted using bold, italics, or different font sizes.
  6. Information is often organized using bullet points or numbered lists.
  7. Forms incorporate various input elements for user data entry.
  8. Specific HTML tags are used to represent each webpage element.
  9. HTML facilitates interactive webpages for data collection and server transmission.
  10. The Paragraph Tag (`<p>`) defines a block of text as a paragraph.

Page 7

  1. Browsers automatically add a line break before and after each paragraph.
  2. The `<p>` tag can contain text, images, links, or other HTML elements.
  3. Attributes can be used to style paragraphs, controlling color or alignment.
  4. Headings (<h1> to <h6>) create structural hierarchy within a webpage.
  5. <h1> represents the main heading, typically the page title.
  6. <h2> to <h6> denote subheadings of decreasing importance.
  7. Best practice: avoid skipping heading levels for logical structure.
  8. Text formatting tags include `<b>` for bold and `<u>` for underlined text.
  9. `<del>` or `<strike>` tags create strikethrough text.
  10. `<sub>` creates subscript text (e.g., H₂O) for scientific content.
  11. `<sup>` creates superscript text (e.g., x²) for mathematical expressions.
  12. Subscript and superscript are common in scientific and mathematical contexts.

Page 8

  1. Hyperlinks are a key HTML feature, enabling navigation between webpages.
  2. The `<a>` tag (anchor) is used to create these hyperlinks.
  3. The `href` attribute within `<a>` specifies the target URL or resource.
  4. Example: `<a href='url'>Visit Example</a>` creates a clickable link.
  5. The `<img>` tag is used to embed images into a webpage.
  6. It requires the `src` (source) attribute to specify the image file location.
  7. Example: `<img src='image.jpg' alt='Sample Image'>` displays an image.
  8. Optional attributes like `width` and `height` control image dimensions.
  9. Images significantly enhance webpage visual appeal and attract user attention.
  10. Common image uses include logos, background visuals, and advertisements.

Page 9

  1. HTML provides unordered (bulleted) lists using the `<ul>` tag.
  2. Ordered (numbered) lists are created using the `<ol>` tag.
  3. Each individual list item is defined by the `<li>` (list item) tag.
  4. Lists can be nested within others to create structured sublists.
  5. Lists are useful for displaying items like shopping lists or process steps.
  6. Tables are used to display structured data, such as price lists.
  7. Tables are created using the `<table>` tag in HTML.
  8. `<tr>` defines a table row, structuring content horizontally.
  9. `<td>` defines a table data cell, holding individual cell content.
  10. `<th>` defines a table header cell, typically bold and centered.
  11. A table structure involves `<tr>` tags containing `<th>` or `<td>` cells.
  12. Forms are the primary method for collecting user input on webpages.

Page 10

  1. Forms allow users to input text, numbers, dates, and upload files.
  2. Users interact with forms using buttons, checkboxes, and radio buttons.
  3. The Google homepage is an example of a simple form with input and buttons.
  4. Forms are constructed using the `<form>` tag in HTML.
  5. Multiple `<input>` tags inside a form specify different user input types.
  6. Common input types include 'text', 'number', 'file', 'radio', 'checkbox', 'submit'.
  7. The `type` attribute defines the specific kind of input field.
  8. The `action` attribute specifies the server URL for processing form data.
  9. Submitted form data is transmitted to the server or stored in a database.
  10. Common forms include login, registration, search, contact, and e-commerce checkout.
  11. Forms enable data collection, personalized experiences, and interactive web functionality.

Page 11

  1. CSS stands for Cascading Style Sheets, controlling HTML element appearance.
  2. CSS defines how content is displayed across various devices like screens and print.
  3. It makes webpages visually appealing with defined colors, fonts, and layout.
  4. A key advantage is achieving consistent design across multiple webpages.
  5. CSS saves developer time by centralizing layout and styling control.
  6. It keeps HTML code clean and organized by separating content from design.
  7. CSS avoids repetition of inline styling, preventing appearance inconsistencies.
  8. Using CSS reduces HTML file size and simplifies website maintenance.
  9. All styles are typically defined in an external CSS file for efficiency.
  10. The `<link>` tag is used to connect the CSS file to HTML pages.
  11. CSS ensures uniformity in elements like buttons, headers, and fonts across a site.

Page 12

  1. A website is a collection of interconnected webpages serving a specific purpose.
  2. Examples include school websites (bmsce.ac.in) and search engines (google.com).
  3. Websites are hosted on servers that handle user requests and content.
  4. Webpages within a site are linked via weblinks for seamless navigation.
  5. A well-designed website is coherent, easy to navigate, and purpose-driven.
  6. Website development involves essential stages: planning, building, and deployment.
  7. Planning, through storyboards and wireframes, is crucial before coding begins.
  8. Storyboards are visual representations of a website's flow and user interactions.
  9. They help identify different user types and their expected interactions.
  10. Storyboards focus on storytelling, navigation flow, and overall user experience.
  11. These planning tools evolved from film and drama to web design.

Page 13

  1. Storyboards serve as the first step before creating detailed wireframes.
  2. Wireframes are simplified visual blueprints showing a webpage's layout.
  3. They illustrate space allocation, element placement, and interaction behavior.
  4. Wireframes are created without colors, fonts, or images to focus on structure.
  5. They are used during the planning phase to detect usability or design issues early.
  6. Wireframes typically use boxes and lines to represent elements like headers and menus.
  7. They outline content sections and calls to action (buttons/links).
  8. Balsamiq is a wireframing tool known for quick prototypes and team feedback.
  9. Figma is an advanced wireframing and collaboration tool (acquired by Adobe).
  10. Figma can directly convert designs into functional HTML websites.
  11. Wireframing tools aid in designing for both web and mobile platforms (iOS/Android).

Page 14

  1. A web server (e.g., Apache) is necessary to host your website files.
  2. HTML pages and related files require an organized folder structure on the server.
  3. Folder structure planning is beneficial during the wireframing stage.
  4. The root folder is the main directory for all website content.
  5. The `index.html` file serves as the default homepage.
  6. Consistent file organization simplifies navigation and future maintenance.
  7. Common folders include `/images`, `/fonts`, `/js`, `/css`, and `/lib`.
  8. An organized structure improves website maintenance and file management.
  9. It also positively impacts SEO (Search Engine Optimization) and user navigation.
  10. A good structure supports Content Management Systems (CMS) like WordPress.

Page 16

  1. Digital cameras led to rapid growth in image editing and manipulation software.
  2. These tools allow removing blemishes, cutting/pasting parts, and creating composites.
  3. Computer graphics is extensively used in film and television for special effects.
  4. Applications include removing support wires and overlaying images digitally.
  5. Graphics applications drove the transition from analog film to digital imaging.

  9. Image Processing analyzes existing images to detect or enhance patterns (AI).
  10. 2D Graphics focuses on creating images; Image Processing focuses on understanding images.

Page 17

  1. 3D Graphics creates 3D virtual scenes and converts them into 2D images.
  2. Unlike real photography, 3D graphics 'photographs' virtual worlds from data.
  3. 3D Graphics creation involves Scene Creation & Encoding and Image Production.
  4. Scene Creation & Encoding is the artistic process of designing the 3D environment.
  5. Image Production is the computational process of generating the final rendered image.
  6. 3D Graphics is widely used in interactive video games and animated movies.
  7. Video games allow players to interact with 3D virtual worlds.
  8. Animators interact with virtual environments to create frames for movies.

Page 18

  1. Future 3D graphics may enable direct interaction with virtual worlds, not just viewing.
  2. Potential exists for 3D graphics projectors and immersive storytelling experiences.
  3. Advances in 3D human-machine interfaces will enable richer interactive virtual worlds.
  4. The 3D graphics process involves three main steps: Modeling, Rendering, Displaying.
  5. Modeling designs and constructs a 3D scene using digitally encoded data.
  6. Modeling is analogous to building a physical set in the movie industry.
  7. The created 3D scene in modeling may not exist in the real world.
  8. Rendering converts the 3D model into a 2D image from a specific camera position.
  9. Rendering applies mathematics and analytic geometry to project objects onto a plane.

Page 19

  1. Perspective Projection projects objects from a viewpoint, mimicking human vision.
  2. Parallel Projection uses parallel projectors, providing true object profiles for engineering.
  3. The Image Window defines the boundaries of the final image on the projection plane.
  4. It acts like a camera viewfinder, displaying only content within its frame.
  5. Pixel computation during rendering is complex, accounting for many factors.
  6. It considers lighting (intensity, direction) and material properties (shiny, transparent).
  7. Pixel computation also accounts for object interactions like shadows and reflections.
  8. This process relies heavily on physics and material science principles.
  9. Computed pixels are stored as a bitmap in a special memory area called the frame buffer.
  10. The frame buffer can reside in main memory or dedicated graphics memory (GPU).

Page 20

  1. The final image in the frame buffer is either displayed or stored.
  2. Display can be immediate (real-time rendering) for games and simulators.
  3. Real-time rendering often limits image quality due to processing constraints.
  4. Animated movies typically achieve higher visual quality than games due to non-real-time rendering.
  5. A video game system represents an encoded 3D virtual world with manipulation software.
  6. As a player interacts, the scene is repeatedly rendered and stored in the frame buffer.
  7. Special-purpose graphics hardware (GPU) handles real-time rendering in games.
  8. The display shows updated images, creating the illusion of smooth movement.
  9. The presence of dedicated graphics hardware distinguishes game consoles from PCs.
