Important Points - Notes

Page 1
  1. Module 5 covers HTML, CSS, Website Development, and Computer Graphics.
  2. Key textbooks include Brookshear & Brylow, and Roy, Daniel & Agrawal.
Page 2
  1. Early Internet (1970s) used plain text and complex commands for researchers.
  2. Sir Tim Berners-Lee invented the World Wide Web (1989) using hyperlinks.
Page 3
  1. HTML developed by Berners-Lee (1993) to easily format and share content.
  2. Mosaic Web Browser (1992-1993) popularized the internet with the <img> tag.
  3. HTML is the standard markup language for creating webpages and applications.
Page 4
  1. HTML uses text-based tags to define webpage structure, content, and layout.
  2. Web browsers interpret HTML code, making pages interactive and dynamic.
  3. Hypertext links connect webpages, forming the foundation of the Internet.
Page 5
  1. HTML pages use tags to instruct browsers how to display text and content.
  2. Basic HTML structure includes DOCTYPE, html tag, head tag, and body tag.
  3. The DOCTYPE declaration specifies the HTML version for the browser.
Page 6
  1. The head tag contains metadata, like title, stylesheets, and JavaScript.
  2. The body tag contains all visible webpage content like text, images, and links.
  3. The <p> tag defines a block of text as a paragraph, adding line breaks.
Page 7
  1. HTML headings (<h1> to <h6>) create structure and hierarchy on a webpage.
  2. Text formatting tags include <b>, <u>, <del>, <strike>, <sub> and <sup>.
Page 8
  1. The <a> tag creates hyperlinks, allowing navigation to other webpages using href.
  2. The <img> tag adds images to webpages, requiring src for the file location.
Page 9
  1. HTML supports unordered (<ul>) and ordered (<ol>) lists, with each item (<li>).
  2. Tables (<table>) display structured data using rows (<tr>), data cells (<td>), and header cells (<th>).
  3. Forms (<form>) collect user input via various <input> types like text or buttons.
Page 10
  1. Input types define fields like text, number, file, radio, checkbox, and submit buttons.
  2. The form's action attribute specifies the server URL for data processing.
Page 11
  1. CSS (Cascading Style Sheets) controls the appearance and layout of HTML elements.
  2. CSS ensures consistent design, saves developer time, and separates content from design.
  3. Using external CSS files centralizes styles, reduces HTML size, and simplifies maintenance.
Page 12
  1. A website is a collection of interconnected webpages serving a specific purpose.
  2. Website development involves planning, building, and deployment, similar to construction.
  3. Storyboards visually represent website flow, user interactions, and overall user experience.
Page 13
  1. Wireframes are simplified visual blueprints of webpage layout, showing element placement.
  2. They help detect usability issues early, focusing on structure without aesthetics.
  3. Tools like Balsamiq and Figma assist in creating wireframes and prototypes.
Page 14
  1. A web server hosts the website, storing HTML files in an organized folder structure.
  2. The root folder's index.html file serves as the default homepage.
  3. Organized file structures improve maintenance, SEO, and user navigation.
Page 15
  1. Website navigation uses subfolders to organize sections, mirroring the site structure.
  2. Computer Graphics uses technology to create and manipulate visual content.
  3. Applications include GUIs, video games, animated movies, and displaying text.
Page 16
  1. Computer graphics encompasses image manipulation, especially in digital photography.
  2. It's used for special effects in film and television, driving digital imaging transition.
  3. 2D graphics tools like Microsoft Paint allow drawing shapes and filling colors.
Page 17
  1. 2D Graphics create images, while Image Processing analyzes existing images.
  2. 3D Graphics creates virtual scenes, then converts them into 2D images.
  3. 3D graphics involves scene creation (artistic) and image production (computational).
  4. Widely used in interactive video games and animated movies for virtual environments.
Page 18
  1. The future of 3D graphics includes immersive interaction with virtual worlds.
  2. 3D graphics paradigm involves Modeling, Rendering, and Displaying.
  3. Modeling designs and constructs a 3D scene using digitally encoded data.
Page 19
  1. Rendering converts 3D models into 2D images, applying mathematics for projection.
  2. Perspective projection mimics human vision; parallel projection shows true object profiles.
  3. Pixel computation during rendering accounts for lighting, material, and object interactions.
  4. The computed pixels are stored as a bitmap in the frame buffer memory.
Page 20
  1. Displaying involves immediate rendering (games) or storing for later (movies).
  2. Real-time rendering prioritizes speed, limiting visual quality compared to movies.
  3. Video game systems use dedicated graphics hardware (GPU) for real-time rendering.
